- 데이터는 보통 데이터베이스 테이블 등에 저장

- Table: 데이터가 저장된 공간

<OLTP>
- 거래를 하기 위해 사용되는 데이터베이스
<SQL>
- 데이터베이스에서 데이터를 가지고 올 때 사용하는 언어
<OLAP>
- 분석을 위한 기능 제공

<BigQuery의 장점>
1. 난이도
2. 속도
3. 데이터 쉽게 추출
4. 데이터 웨어하우스를 사용하기 위해 서버를 띄울 필요 없음

## 문제 2

> **🧚Q. 이번 강의를 통해 SQL이 왜 필요하다고 느끼는지, SQL을 통해 본인이 어떤 것을 해내고 싶은지를 자유롭게 작성해보세요.**

이번 강의를 통해 SQL은 대규모 데이터를 효율적으로 관리하고 필요한 정보를 빠르게 추출하기 위해 필요한 도구라고 느꼈다. 데이터가 많아질수록 원하는 정보를 일일이 확인하는 것은 어렵기 때문에, SQL을 활용해 데이터를 체계적으로 조회하고 분석하는 능력이 중요하다고 생각했다.

특히 HR 분야에서도 SQL은 유용하게 활용될 수 있다고 느꼈다. 예를 들어 직원들의 교육 이수 기록, 성과 데이터, 근속 기간, 만족도 조사 등의 데이터를 SQL을 통해 정리하고 분석한다면 조직의 인재 관리나 교육 프로그램 개선에 필요한 인사이트를 얻을 수 있을 것이다.

앞으로 SQL을 활용해 다양한 데이터를 기반으로 조직의 문제를 분석하고 해결하는 데 기여하고 싶다. 나아가 HR 업무를 수행할 때 사람과 조직을 이해하는 것뿐만 아니라 데이터를 통해 의사결정을 지원할 수 있는 역량을 갖추고, 데이터까지 아우르는 HR 전문가로 성장하고 싶다.
